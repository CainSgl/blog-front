# PWA 截图准备指南

## 概述

为了让 PWA 在安装时显示更丰富的安装 UI，需要提供应用截图。这些截图会在用户安装应用时展示，帮助用户了解应用的功能和界面。

## 截图要求

### 移动端截图（form_factor: narrow）

- **尺寸要求**：
  - 最小宽度：320px
  - 最大宽度：720px
  - 最小高度：320px
  - 最大高度：1440px
  - 宽高比：建议 9:16 或类似移动设备比例
  - 推荐尺寸：540x720、360x640、375x667

- **数量**：至少 1 张，建议 2-5 张

- **内容建议**：
  - 首页/主界面
  - 核心功能展示
  - 知识库浏览界面
  - 文章阅读界面
  - 用户个人中心

### 桌面端截图（form_factor: wide）

- **尺寸要求**：
  - 最小宽度：1024px
  - 最大宽度：3840px
  - 最小高度：593px
  - 最大高度：2160px
  - 宽高比：建议 16:9 或类似桌面显示器比例
  - 推荐尺寸：1280x720、1920x1080

- **数量**：至少 1 张，建议 2-4 张

- **内容建议**：
  - 桌面端首页全景
  - 知识库管理界面
  - 文章编辑器界面
  - 用户管理界面

## 截图制作步骤

### 方法 1：使用浏览器开发者工具

1. 打开你的应用
2. 按 F12 打开开发者工具
3. 点击设备模拟按钮（或按 Ctrl+Shift+M）
4. 选择设备类型：
   - 移动端：iPhone 12 Pro (390x844)
   - 桌面端：设置为 1280x720 或更大
5. 截取屏幕（可以使用浏览器的截图功能或系统截图工具）

### 方法 2：使用在线工具

- [Screely](https://www.screely.com/) - 为截图添加漂亮的背景
- [Mockuphone](https://mockuphone.com/) - 将截图放入设备框架中
- [Figma](https://www.figma.com/) - 专业的设计工具

### 方法 3：实际设备截图

- 在真实的移动设备或桌面浏览器中打开应用
- 使用系统截图功能
- 确保截图尺寸符合要求

## 文件组织

将截图文件放在 `public/screenshots/` 目录下：

```
public/
└── screenshots/
    ├── mobile-1.png      # 移动端首页
    ├── mobile-2.png      # 移动端知识库
    ├── mobile-3.png      # 移动端文章阅读（可选）
    ├── desktop-1.png     # 桌面端首页
    ├── desktop-2.png     # 桌面端知识库
    └── desktop-3.png     # 桌面端编辑器（可选）
```

## 截图优化建议

1. **清晰度**：使用高质量的截图，避免模糊
2. **内容**：展示应用的核心功能和特色
3. **真实性**：使用真实的内容，而不是占位符
4. **一致性**：保持截图风格统一
5. **文件大小**：优化图片大小，建议每张不超过 500KB
6. **格式**：推荐使用 PNG 或 WebP 格式

## 图片压缩

使用以下工具压缩截图：

- [TinyPNG](https://tinypng.com/)
- [Squoosh](https://squoosh.app/)
- [ImageOptim](https://imageoptim.com/)（Mac）

## 配置说明

截图已在 `vite.config.js` 中配置：

```javascript
screenshots: [
  {
    src: '/screenshots/mobile-1.png',
    sizes: '540x720',
    type: 'image/png',
    form_factor: 'narrow',
    label: '首页展示'
  },
  // ... 更多截图
]
```

## 测试

1. 运行 `npm run build` 构建应用
2. 运行 `npm run preview` 预览
3. 在 Chrome 中打开应用
4. 打开开发者工具 > Application > Manifest
5. 查看 "Screenshots" 部分，确认截图正确加载

## 注意事项

- 如果暂时没有准备好截图，可以先注释掉 `screenshots` 配置
- 截图不是 PWA 安装的必需项，但会提供更好的用户体验
- 不同浏览器对截图的支持程度不同，主要是 Chrome/Edge 支持

## 临时解决方案

如果你现在不想添加截图，可以暂时注释掉配置中的 `screenshots` 部分，这样就不会有警告了。
